Programming/
ООП Объектно-ориентированное программирование
Object-oriented programming
SOLID Пять принципов ООП, благодаря которым программисты могут упрощать свой код, избегая сложных и запутанных конструкций
DDD Domain-Driven Design — Подход к разработке программного обеспечения, ориентированный на модель предметной области
MVP Minimum Viable Product (Минимально жизнеспособный продукт)
REST Representational State Transfer
Парадигмы программирования Императивная (КАК)
Декларативная (ЧТО)
Interpreter Compiler Интерпретатор — Программа, которая читает и исполняет команды на другом языке программирования
Компилятор — Программа, которая преобразует код на одном языке программирования в тот что может быть понят целевой машиной
sync async Синхронность — Означает, что программа выполняет одну задачу за раз
Асинхронность — Означает, что программа может выполнять несколько задач одновременно
ip (от англ. Internet Protocol) — уникальный числовой идентификатор устройства в компьютерной сети, работающей по протоколу IP
Big O Notation Математическая нотация, которая описывает поведение алгоритмов с точки зрения их асимптотической сложности
ECS Entity-Component-System — это архитектура, где сущности — это ID, компоненты содержат только данные, а системы реализуют поведение, обрабатывая сущности с нужными компонентами
