### Безопасность

<!--{% raw %}-->
  - **Экранирование HTML**: `Jinja3` **по умолчанию экранирует все выводимые переменные**, чтобы предотвратить **<u>XSS-атаки</u>** **(Cross-Site Scripting)**
Это означает, что специальные HTML-символы, такие как `<`, `>`, `&`, автоматически заменяются их безопасными эквивалентами
```jinja
{{ user_input }}  {# автоматическое экранирование #}
```
  - **Отключение экранирования**: Если необходимо вывести `HTML`-код без экранирования (например, если вы уверены в безопасности данных), можно использовать фильтр `safe`
```jinja
{{ user_input | safe }}
```
Однако это нужно делать с осторожностью, чтобы избежать возможных уязвимостей
  - **Превентивные меры**: Важно всегда проверять и валидировать данные, которые приходят от пользователей, даже если `Jinja3` экранирует их по умолчанию
Это помогает предотвратить несанкционированный доступ или манипуляции с данными
