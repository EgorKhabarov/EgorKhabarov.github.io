# Шпаргалка
```
Напиши максимально подробную и структурированную шпаргалку по теме, которую я тебе напишу
Используй таблицы и код если этого требует тема
```

```
Подробно расскажи про <...>

Оформи в виде шпаргалки
Не используй --- для разделения категорий
Не используй эмодзи
Не делай заголовки и заголовки таблиц bold
Не делай нумерацию заголовков типа `### 1.`
Не используй код в заголовке типа `### \`code\``
Вместо длинного тире всегда используй `-`

Используй заголовки так:
# - Для основных частей шпаргалки
## - Для подтем
### - Для примеров кода (если необходимо)
#### - Для разделения примеров кода (если необходимо несколько подпримеров кода внутри одного ### заголовка)

Для markdown списков используй `-`
В таблицах не оставляй пробелы и например вместо `| --- |` пиши `|-----|`
Если это название функции класса или метода или что то похожее то предпочтительно обрамлять кодом `\``

Предпочитительно использовать двойные кавычки `"` там где это не меняет смысла
Исключение: языки в которых одинарные кавычки `'` прописаны в style guides (SQL) или меняют смысл (C|C++|Java char)

Не используй точки в конце параграфа (после предложений ставь, а после последнего предложения в параграфе не ставь)
Не оставляй в конце строк лишние символы пробела
Не пиши `Пример` перед примером кода или `Результат` перед результатом и не делай лишних очевидных пометок

Обязательно используй заголовок # в самом начале шпаргалки с кратким названием шпаргалки
```
